**Effort Level:** Medium-High

**Skills Demonstrated:** Judgment, discretion, regulatory awareness, business context, escalation decisions

---

## Background

### What Is Communications Review?

Communications review is a critical compliance function where analysts:
- **Review** alerts generated by surveillance systems
- **Apply judgment** to determine if flagged communications warrant escalation
- **Analyze** sophisticated communications by understanding business context
- **Identify** potential regulatory concerns
- **Document** decisions with clear rationale

### Why This Role Matters

In regulated industries (especially financial services), electronic communications surveillance is legally required. Reviewers serve as the human layer between automated systems and formal investigations.

| Responsibility | Impact |
|---------------|--------|
| Identify genuine concerns | Protect firm from regulatory violations |
| Filter false positives | Enable business to operate efficiently |
| Understand context | Prevent over-escalation of legitimate activity |
| Document thoroughly | Support audit and regulatory examination |
| Escalate appropriately | Ensure serious matters get proper attention |

### Real-World Context

This exercise simulates the daily work of a Global Communications Review Specialist. You'll review a queue of flagged communications, each requiring:
- Context analysis
- Regulatory knowledge
- Business understanding
- Sound judgment
- Clear documentation

---

## Learning Objectives

By completing this exercise, you will be able to:

1. **Review** flagged communications with appropriate skepticism and context
2. **Identify** regulatory red flags in various communication types
3. **Distinguish** between genuine concerns and false positives
4. **Document** review decisions with proper rationale
5. **Escalate** appropriately based on issue type and severity
6. **Apply** judgment in ambiguous situations

## Key Concepts

### The Reviewer Mindset

```
┌─────────────────────────────────────────────────────────────────┐
│                    REVIEW APPROACH                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. READ THE FULL CONTEXT                                       │
│     Don't react to a single message - read the whole thread     │
│                                                                  │
│  2. UNDERSTAND WHO IS INVOLVED                                  │
│     Roles, relationships, and motivations matter                │
│                                                                  │
│  3. CONSIDER BUSINESS CONTEXT                                   │
│     What's happening in the market? Any known events?           │
│                                                                  │
│  4. APPLY REGULATORY KNOWLEDGE                                  │
│     What rules might apply? What are the red flags?             │
│                                                                  │
│  5. USE JUDGMENT, NOT JUST KEYWORDS                             │
│     Lexicons flag possibilities - you determine reality         │
│                                                                  │
│  6. DOCUMENT YOUR REASONING                                     │
│     Future reviewers should understand your decision            │
│                                                                  │
│  7. WHEN IN DOUBT, ESCALATE                                     │
│     It's better to over-escalate than miss something            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Common Review Dispositions

| Disposition | When to Use |
|-------------|-------------|
| **No Issue Identified** | Clear business context explains the flag |
| **False Positive** | Lexicon triggered but clearly not concerning |
| **Needs More Context** | Cannot determine without additional information |
| **Monitor/Note** | Minor concern, document for pattern detection |
| **Escalate to Compliance** | Potential policy or regulatory issue |
| **Escalate to Legal** | Potential legal exposure or litigation hold |
| **Immediate Escalation** | Urgent matter requiring same-day attention |

3. How do you handle communications in languages you don't speak?

5. How do you protect confidentiality while doing your job?
