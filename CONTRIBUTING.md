# Contributing to Compliance Frameworks

Thank you for your interest in contributing! This repository aims to be a comprehensive, practical resource for compliance professionals at all levels.

---

## Ways to Contribute

### 1. Report Issues
- Found an error in a framework description?
- Spotted outdated information?
- Have a suggestion for improvement?

[Open an issue](../../issues/new/choose) and let us know.

### 2. Add Content
- New glossary terms
- Additional framework cheat sheets
- Industry-specific examples
- Interview questions and scenarios

### 3. Improve Existing Content
- Fix typos or unclear wording
- Update outdated regulatory information
- Add practical examples
- Enhance templates

### 4. Create New Exercises
- Scenario-based learning exercises
- Industry-specific case studies
- Real-world compliance challenges

## Contribution Guidelines

### Content Standards

| Guideline | Description |
|-----------|-------------|
| **Accuracy** | Verify regulatory information against official sources |
| **Practical** | Focus on actionable knowledge, not theory |
| **Industry-agnostic** | Keep core content applicable across industries |
| **Beginner-friendly** | Explain jargon, don't assume knowledge |
| **Concise** | Respect readers' timeâ€”be thorough but not verbose |

### What We're Looking For

**Good contributions:**
- Add practical value for compliance professionals
- Are well-researched and accurate
- Follow existing formatting conventions
- Include examples where helpful

**We'll likely decline:**
- Marketing content or vendor promotion
- Overly theoretical academic content
- Region-specific content without clear labeling
- Content that duplicates existing material

## How to Submit Changes

### For Small Changes (typos, minor fixes)

1. Fork the repository
2. Make your changes
3. Submit a pull request with a clear description

### For Larger Changes (new content, significant updates)

1. **Open an issue first** to discuss your proposed changes
2. Wait for feedback before investing significant time
3. Fork the repository
4. Create a branch for your changes
5. Make your changes following our standards
6. Submit a pull request referencing the issue

---

## Pull Request Process

1. **Describe your changes** - What did you add/change and why?
2. **Reference any issues** - Link to related issues
3. **Check formatting** - Ensure markdown renders correctly
4. **Verify accuracy** - Double-check regulatory information
5. **Be responsive** - Address feedback promptly

### PR Title Format

```
[Type] Brief description

Types:
- [Fix] - Correcting errors
- [Add] - New content
- [Update] - Improving existing content
- [Docs] - Documentation improvements
```

Examples:
- `[Fix] Correct GDPR breach notification timeline`
- `[Add] ISO 27001 cheat sheet`
- `[Update] Expand SOC 2 glossary terms`

## Content Formatting

### Markdown Standards

- Use ATX-style headers (`#`, `##`, `###`)
- Use tables for structured information
- Use code blocks for examples
- Add horizontal rules (`---`) between major sections

### File Naming

- Use lowercase with hyphens: `framework-name.md`
- Be descriptive: `hipaa-breach-notification.md` not `breach.md`

### Glossary Entries

When adding glossary terms:

```markdown
| **Term** | Definition | Example |
```

- Keep definitions concise (1-2 sentences)
- Include practical examples
- Add to the appropriate category section

### Exercise Format

New exercises should include:

1. **README.md** - Overview, learning objectives, instructions
2. **Scenario files** - The situation to analyze
3. **Solutions** - Expected approach (not just answers)
4. **Additional context** - Supporting materials as needed

---

## Regulatory Updates

Compliance regulations change. When updating regulatory information:

1. **Cite your source** - Link to official regulatory text
2. **Note the effective date** - When did this change take effect?
3. **Update all affected files** - Check glossary, cheat sheets, exercises
4. **Consider practical impact** - How does this change affect practitioners?

## Code of Conduct

### Our Standards

- Be respectful and constructive
- Focus on content, not personalities
- Welcome newcomers and help them contribute
- Accept constructive criticism gracefully

### Unacceptable Behavior
